2. Methodology - Algorithms

1. Collaborative Filtering (KNN-Based)

Finds similar users/items based on rating patterns using K-Nearest Neighbors

  • KNNBasic: Uses cosine similarity to find nearest neighbors
  • KNNWithMeans: Accounts for user rating biases by centering ratings
  • Similarity Metrics: Cosine, Pearson correlation

2. Matrix Factorization (SVD)

Decomposes user-item rating matrix into latent factors

  • Factors: 100 latent features
  • Learning Rate: 0.005
  • Regularization: 0.02 (prevents overfitting)

3. Content-Based (TF-IDF)

Uses product text features for similarity matching

  • Combines: Product name + Category + Reviews
  • Max features: 5000 terms
  • Similarity: Cosine similarity on TF-IDF vectors

2. Methodology - System Workflow

Step Process Description
1 Data Loading Load Amazon Products CSV + Reviews text file
2 Data Combining Generate synthetic User/Product IDs, assign reviews
3 Data Cleaning Handle missing values, fix price logic, convert types

5 EDA Generate visualizations, insights, feature analysis
6 Train-Test Split 80/20 split for model evaluation
7 Model Training Train 4 models: Baseline, KNN, SVD, KNNWithMeans

9 Model Selection Select best model based on RMSE/MAE

3. Module Description (Continued)

ğŸ¯ Module 4: Inference & Recommendation Engine

KNN Recommender

  • Uses trained KNN model similarity matrix
  • Finds top-N similar products
  • Returns product details (name, image, link)

TF-IDF Recommender

  • Content-based fallback
  • Uses product text similarity
  • Handles cold-start problem

Hybrid Recommender Strategy

Primary: KNN-based collaborative filtering
Fallback: TF-IDF content-based (when KNN returns < 3 results)
Output: Top 6 product recommendations with metadata
